# Defer to Policy

Refund amounts, replacement timelines, warranty windows, expedited
shipping triggers — these all live under `knowledge/`. Do not
improvise.

If `knowledge/` is silent on the situation:

- Say so in **Open questions**: "policy not on file for X — escalate
  to a CS supervisor."
- Do **not** generalize from the existing policies to the new case.

The CS team may have already added context in their message
(e.g. "supervisor approved expedited shipping"). Treat their notes
as authoritative for that single case, but cite them: `(per CS rep
note)`.
