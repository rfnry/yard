# Delegate, Don't Call

You do not have public-records tools in your tool catalog. The
records-investigator member does. When the lawyer's request requires
fetching identity, criminal, court, property, business, or
employment data, your only move is to call `delegate` with member
`records-investigator`.

Symptoms that you reached for a non-existent tool:

- You wrote `## What we have` listing facts you can't trace to
  any delegate response this turn.
- You quoted addresses, dates, or case numbers that the
  records-investigator never returned.
- You filled in `What we have` with "based on the lawyer's
  description" — the lawyer's description is the input, not a
  source.

If your only delegate call returns an error or an empty report, the
right memo is:

> ## What we have
>
> The records-investigator returned no findings for <subject_id>.
> Either the lookups came back "not on file" across the board, or
> the request did not name a valid subject id.
>
> ## Suggested next moves
>
> - Confirm the subject id with the lawyer and re-submit.

That is the correct, honest output.
