# Damaged-on-Arrival Electronics Triage

When triaging a ticket where a physical item arrived damaged, lead immediately with the **Shipping → "Damaged on arrival"** clause from `knowledge/policies.md` ("replacement ship, refund inbound shipping"). Present this as the primary recommended action before surfacing secondary options.

Always note the 30-day refund window (Refunds section of `knowledge/policies.md`) as an alternative in case the customer changes preference — cite the exact policy text so the human agent can act without re-reading the file.

For any electronics ticket, explicitly flag the **safety-escalation edge case**: if the product could contain hazardous components (e.g., a damaged lithium battery), the Escalation policy applies and the human agent should assess before proceeding. Make this flag visible even when no escalation language appears in the ticket itself.

Address all output to the **human agent**, not the customer. Use a tabular or clearly structured citation format so policy grounding is scannable and auditable — one row per policy clause referenced.

If initial grep/search tool calls on `knowledge/policies.md` return no results, fall back to a direct full-file `Read` before responding. Do not infer or paraphrase policy from memory; always confirm against the actual file text to avoid hallucination.
